## Notes

- Open Data Portal uses ArcGIS Hub (portal-chemnitz.opendata.arcgis.com); FeatureServer endpoint: https://services6.arcgis.com/jiszdsDupTUO3fSM/arcgis/rest/services/Wahlbezirke_FL_1/FeatureServer/0
- Live feature count verified 2026-05-04: 129 polygons. WAHLBEZIRK codes are 4-digit (e.g. 0104, 8204, 8702, 9203); STADTTNAME labels Stadtteil. Schema: OBJECTID, ID, WAHLBEZIRK, STADTTS, STADTTNAME + shape area/length.
- Discrepancy with task brief: brief expected ~170 WBZ but live layer has 129. May reflect (a) recent consolidation (city had ~170 Wahlbezirke in earlier elections), or (b) only Urnenwahlbezirke without sub-splits. Worth confirming with Wahlamt before relying on counts for BTW analysis.
- Dataset created 2015-12-04, first published as Open Data 2021-12-02, last edited 2025-06-10. Update frequency "nach Bedarf" — boundaries updated for each election but old versions are NOT separately published. Only current snapshot is available.
- For 2013/2017/2021 historical Wahlbezirk geometries, primary path is a request to gdi@stadt-chemnitz.de (technical) cc a18@stadt-chemnitz.de (substantive) and wahlbehoerde@stadt-chemnitz.de (election authority). The city's internal GIS likely retains historical layers; FragDenStaat could be used as fallback.
- Sachsen state geoportal (geoportal.sachsen.de) hosts only Verwaltungsgrenzen (LK/Gemeinde), not municipal Wahlbezirke. govdata.de mirrors the Chemnitz Open Data record but adds no historical versions.
- Companion statistical portal chemnitz-in-zahlen.de offers election results aggregated to Stadtteile/Wahlbezirke and downloadable PDF map books, but no shapefiles.
- "Stadtteile" layer (39 polygons) is a separate dataset and is not the WBZ layer; brief's caveat "not 39 Stadtteile" is correctly handled — 129 is the WBZ-level layer.
