## Notes

- ArcGIS REST endpoint serves GeoJSON via `/query?where=1=1&outFields=*&f=geojson` (paged at 1000). Sample attributes: `WB` (4-char Wahlbezirk code, e.g. "0111"), `SB` (Stimmbezirk), `OBZ`/`OBZ_Name` (Ortsbezirk = Stadtteil, 23 of these — used as covariate, not as WBZ).
- 147 polygons confirms WBZ granularity (much finer than the 23 Stadtteile). Search hit "Ortsbezirke (Stadtteile)" is layer 0 — do NOT use that one.
- No temporal/year attribute, no archive layers visible via REST search (only Wahlen WebMap, Wahlen WebApp). Historical 2013/2017/2021 boundaries are not exposed publicly; must request from Kundenzentrum Geodaten.
- govdata.de and opendata.hessen.de did not list a Kassel WBZ dataset; geoHub at datenkatalog-kassel-geoportal.hub.arcgis.com is the canonical catalog but Wahlbezirke is published only as the live REST layer (no item page for download).
- Statistikstelle compiles BTW results by Ortsbezirk and Wahlbezirk (PDF reports) but does not publish polygons; ekom21 votemanager exports only CSV (tabular).
- Recommended ask to Kundenzentrum: WBZ polygons used for BTW 2013, 2017, 2021, 2025 as Shapefile/GPKG with `WB`-code attribute and election-date metadata.
