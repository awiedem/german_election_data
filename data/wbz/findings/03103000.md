## Notes

- Verified: zip downloaded (50,817 B), 5 files, .prj = ETRS_1989_UTM_Zone_32N (EPSG:25832), .shx record count = 103 polygons. DBF inspected with dbfread: fields ID(N), WAHL(C), WAHLBEZIRK(N); all 103 rows carry WAHL = "Bundestagswahl 2025-02-23". This is the Urnen-WBZ layer (~70-100 expected for a Großstadt of ~125k pop), not the 16 Ortsteile and not the 40 statistische Bezirke.
- Wolfsburg's geonetwork lists two records: 208cdcda... (Wahlbezirke Darstellungsdienst, WMS) and a7338acd... (Wahllokale, point dataset, separate). Wahlbezirke direct download (zip) is not advertised in either Geonetwork record — found via metadata path /metadaten/geodaten/wahlbezirke/wob_wahlbezirke.zip (parallels published wob_wahllokale.zip pattern) and confirmed by WFS GetCapabilities exposing SHAPE-ZIP. Stable enough to script.
- No BTW 2013 / 2017 / 2021 archived editions are published anywhere on geoportal.stadt.wolfsburg.de, the NI geoportal record (397b29df-...), govdata.de, or Google. The WAHL attribute hard-codes the current election only, so historical boundaries must be requested by email.
- Recommended outreach order: geoportal@stadt.wolfsburg.de (primary — holds the file & layer history), cc wahlen@stadt.wolfsburg.de (Wahlamt — knows which Einteilung was used per BTW), LGLN as fallback only.
- Wahlergebnisse-Frontend (wahlen.wolfsburg.de) is a results presentation only — no geometry downloads.
