## Notes

- All four BTW years are effectively covered by three published Stimmbezirk shapefiles (2014/2020/2025 vintages). Each layer has exactly 323 features — matches "300+ WBZ" expectation; clearly Stimmbezirke, not 7 Stadtbezirke or 46 Ortsteile / Kommunalwahlbezirke (those live in separate datasets: opendata-duisburg.de/dataset/ortsteile and /dataset/kommunalwahlbezirke).
- All three zips return HTTP 200; verified locally — sizes 415–423 KB, contain valid .shp/.shx/.dbf/.prj. CRS in .prj is set; not inspected here but presumably ETRS89/UTM32 (EPSG:25832), the NRW standard.
- For BTW 2013 there is no dedicated layer; the "2014" file is the earliest available and the portal's own description groups it as "2014-2019". To use it for BTW 2013, cross-check STB IDs in BTW 2013 result CSVs (https://wahlergebnis.duisburg.de/) against the 323 STB values in the shapefile and flag any unmatched.
- DBF attributes in 2025 layer: STB (text), KWBtxt, KWBnum, BW, Briefwahl, create_ (date). 2020 and 2014 layers use field name STB (matches Stimmbezirk-ID used in result CSVs).
- Alternative sources (not needed but available): ArcGIS REST JSON and KMZ endpoints linked from each opendata-duisburg.de dataset page; geoportal.ruhr metadata record c884eda6-442b-43a3-8d77-f5e24f1b237f catalogs all four Wahlgebiete datasets (Stimmbezirke, Landtagswahlkreise, Bundestagswahlkreise, Kommunalwahlbezirke).
- No WFS endpoint for Stimmbezirke was discoverable in this search; the city's geoportal.duisburg.de viewer exposes layers but no per-Stimmbezirk WFS was advertised in dataset metadata.
- Email format on duisburg.de pages is obfuscated ("wahlamtstadt-duisburgde"); the standard institutional pattern is `<local>@stadt-duisburg.de`, confirmed for `gis@`, `kataster@`, and the named contact `d.mueller-warnecke@stadt-duisburg.de`.
