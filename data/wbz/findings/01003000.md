## Notes

- Lübeck is open-data forward: WBZ layer published as CC BY 4.0 GeoJSON via opendata.luebeck.de and harvested into opendata.schleswig-holstein.de. Verified 111 features (level=Wahlbezirk, props: key/num/name/level/uri).
- Layer is a single current snapshot, not per-election. Metadata `temporal_start=2017-05-07`, `temporal_end=2025-07-18` — implies stability across BTW17/BTW21 but not guaranteed.
- For BTW25 specifically, prefer the Statistik Nord shapefile (election-specific cut, includes Briefwahl-Stimmbezirke logic). HEAD checks return HTTP 200.
- For BTW13 and earlier election-specific cuts, no public archive found. FragDenStaat IFG request from 2023 to Bereich 1.102 successfully returned `luebeck-wahlbezirke-2016.zip` (47 KB) — same channel should work for 2013/2017/2021 historical snapshots: https://fragdenstaat.de/anfrage/geodaten-der-wahlbezirke-und-wahlkreise-der-hansestadt-luebeck/
- Lübeck Geoportal (https://geoportal.smart-hl.city/) and Smart City CKAN (https://platform.smart-hl.city/opendata) exist but the dataset listings 404 directly; SH portal API is the reliable route.
- Sibling Lübeck geometry layers: Stadtbezirke, statistische Bezirke, Sozialbezirke, Landtagswahlkreise, Kommunalwahlkreise — 111-feature WBZ layer is the right one (not 10 Stadtteile / 5 Stadtbezirke).
