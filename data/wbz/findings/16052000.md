## Notes

- Gera publishes only two Geobasisdaten datasets on opendata.gera.de: `Adressen` (CSV/SHP) and `Strassenverzeichnis` (PDF). Other listed datasets cover Ortsteile, Statistische Bezirke, schools, kindergartens, playgrounds — none are Wahl-related.
- Search for "wahl" on opendata.gera.de explicitly returns "Keine Datensätze gefunden".
- The Stadt-Gera GeoPortal at geoportal.gera.de runs on GajaMatrix (JS-rendered); featured maps include Stadtmauer, Ärzte/Apotheken, Busparkplätze, SmartCity, Verkehrsinfo, Spielplätze, Adressen, historical maps. No public Wahlbezirk layer was surfaced via search; a deeper interactive inspection or WMS/WFS GetCapabilities probe (not done here) might still expose an internal layer.
- Thüringen state line (StaLA / wahlen.thueringen.de): per FragDenStaat responses, the Landeswahlleitung explicitly confirms it holds no WBZ geodata — division is municipal responsibility. So state-level fallback is not viable for WBZ polygons.
- Thüringer Landesamt für Bodenmanagement und Geoinformation (TLBG) handles Liegenschaftskataster for Gera (Zweigstelle Zeulenroda) but does not publish municipal WBZ either; included only as cadastral fallback if Stadt Gera cannot supply.
- Recommended outreach order: (1) statistik@gera.de — they run both the OpenData portal and the Wahlamt-adjacent Statistik function, so a single mail can request all four BTW vintages; (2) Sven Gruner (gruner.sven@gera.de) for technical Geoinformation follow-up; (3) Wahlbüro by phone (0365 838-1399) if email request stalls. Expected ~60 WBZ matches Stadt-Gera's typical Bundestagswahl polling-district count (Wahlkreis 193 has ~190k eligible voters across Gera + Greiz + Altenburger Land, of which Gera contributes roughly a third).
