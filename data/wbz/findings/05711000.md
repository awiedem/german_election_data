## Notes

- Bielefeld is a mature open-data publisher: Stimmbezirke layer is offered as Shapefile, CSV, WFS, WMS via the Open Data portal under CC BY 4.0. Direct WFS endpoint base: https://www.bielefeld01.de/md/WFS/wahlen/02?
- Verified feature counts via WFS `RESULTTYPE=hits`: stimmbezirke_pl = **184** polygons (Urnen-Stimmbezirke for BTW/EP/LTW/Kommunal); wahlbezirke_pl_view = **33** (council Wahlbezirke — NOT what is wanted). The task brief's "~280 WBZ" estimate is too high; the actual Stimmbezirk count is 184.
- Only the *current* (2024-12-11) version is published. No 2013/2017/2021 snapshots are listed on Open Data Bielefeld, Open.NRW, govdata.de, or the Geoportal. A precedent exists: the 2024 EP request was fulfilled at https://open-data.bielefeld.de/node/3827, suggesting historical years can likely be obtained on request.
- For BTW, Bielefeld reuses the Kommunalwahl Stimmbezirk-Einteilung — i.e. one geometry layer drives BTW/EP/LTW. Boundary changes between elections are typically minor; Wahlteam should hold archived versions.
- Recommended outreach: primary = wahlteam@bielefeld.de (operational owner of the layer per dataset metadata) with cc to geoinformationundkataster@bielefeld.de (technical custodian) and opendata@bielefeld.de (portal manager). Ask explicitly for archived Stimmbezirk-Geometrien for BTW 2013, 2017, 2021.
- Bielefeld lies in Wahlkreis 132 (BTW); ~280 polygons would be the count if one mistakenly conflated Urnen + Briefwahl-Stimmbezirke.
