## Notes

- Bonn does NOT archive per-election polygon snapshots: each Bundestagswahl dataset on opendata.bonn.de (2013, 2017, 2021, 2025) ships only CSVs (`opendata-wahllokale.csv`, `opendata-strassen.csv`, results) and points to a single, always-current `Flächen der Wahlbezirke Stimmbezirke` layer for geometries.
- Boundary changes are documented: a major reorganisation took place ahead of the 2013 Bundestagswahl (Stimmbezirke combined/recreated, "comparisons only meaningful at Wahlbezirk level"), and again in Feb 2020 for the Kommunalwahl (33 WBZ design with ±15% population balance, used since BTW 2021). The current 33-WBZ layer is therefore identical between BTW 2021 and BTW 2025; the 2017 boundaries differ.
- For BTW 2013/2017/2021 polygon retrieval, best path is (a) Wayback Machine snapshots of `https://stadtplan.bonn.de/geojson?Thema=19411` taken close to each election date, or (b) direct request to `opendata@bonn.de` / `gbkundenzentrum-geodaten@bonn.de` for archived shapefiles. Bonn's open-data team is responsive and the data is CC0, so an email request is low-friction.
- Verified WBZ granularity: 33 Wahlbezirke (BTW level) / 163 Stimmbezirke (Kommunalwahl polling-district level) — not the 4 Stadtbezirke or 51 Ortsteile. Note: ~230 WBZ in the task brief overstates Bonn's count; actual is ~163 Stimmbezirke under the post-2020 design.
- Single endpoint `stadtplan.bonn.de/geojson?Thema=19411` (older API form) and `?OD=4873/4874` (newer DKAN-resource form) both point to the same live layer.
- ArcGIS Hub mirror exists (`hub.arcgis.com/datasets/esri-de-content::wahlbezirke-bonn`) — same data, also CC0.
- Primary geodata authority is the *Amt für Bodenmanagement und Geoinformation* (Kundenzentrum Geodaten); however, the actual publisher on the open-data portal is the Stadt Bonn OpenData team — for this dataset, contacting `opendata@bonn.de` is more direct than the Vermessungsamt.
